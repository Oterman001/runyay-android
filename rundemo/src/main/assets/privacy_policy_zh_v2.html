第一条 定义及适用范围

1.1 定义

  • 跑鸭·RunYay：指跑鸭·RunYay APP 及其提供的全部相关服务，由成都好多鸭科技有限公司开发运营。
  • 个人信息：指以电子或其他方式记录的，能够单独或与其他信息结合识别特定自然人身份或反映特定自然人活动情况的各种信息，包括但不限于自然人的姓名、出生日期、身份证件号码、个人生物识别信息、住址、电话号码、电子邮箱地址、健康信息、行踪轨迹等。
  • 敏感个人信息：指一旦泄露或者非法使用，容易导致自然人的人格尊严受到侵害或者人身、财产安全受到危害的个人信息，包括但不限于生物识别、宗教信仰、特定身份、医疗健康、金融账户、行踪轨迹等信息，以及不满十四周岁未成年人的个人信息。本政策中将以粗体对敏感个人信息进行显著标识。
  • 儿童：指不满十四周岁的未成年人。
  • 系统权限：指移动智能终端操作系统向 App 开放的、具有收集个人信息能力的系统权限。您在使用跑鸭·RunYay 部分业务功能时，需要授权 App 调用相应的系统权限，具体权限说明详见本条第 1.3 节及第二条第 2.5 节。

1.2 适用范围

本隐私政策适用于跑鸭·RunYay 移动应用程序（iOS 版及 Android 版）及其提供的所有相关服务。

除本政策说明的相关信息收集使用活动外，本政策不适用于本平台上接入的其他第三方主体向您提供的其他服务，其他服务适用其向您另行说明的隐私政策。

1.3 可能调用的系统权限清单

根据功能需要，我们的 App 可能调用以下系统权限（并非全部权限均在所有版本或所有设备上使用）：

  • 位置权限（精确位置 / GPS）——iOS：定位服务；Android：ACCESS_FINE_LOCATION
  • 健康/运动数据权限——iOS：HealthKit；Android：运动健身权限（ACTIVITY_RECOGNITION）
  • 相机权限——拍照记录或更换头像
  • 相册/存储权限——iOS：相册访问；Android：READ_EXTERNAL_STORAGE / WRITE_EXTERNAL_STORAGE / READ_MEDIA_IMAGES
  • 蓝牙权限——连接外部运动设备（心率带、智能手表等）
  • 传感器权限——加速度计、陀螺仪、气压计、磁力计（用于运动数据计算）
  • 通知权限——系统消息推送
  • 网络权限——数据上传下载
  • 麦克风权限——未来语音交互功能（使用前将再次请求授权）
  • 后台运行/省电白名单（Android）——保障跑步过程中 App 持续运行

您可在"系统设置 → 应用 → 跑鸭 → 权限"（Android）或"系统设置 → 隐私与安全"（iOS）中随时查看和管理上述权限。关闭某项权限将导致对应功能不可用，但不影响其他功能的正常使用。

2.4 各功能场景信息收集说明

（一）用户注册与登录

必要信息：注册时，您需要提供手机号码，我们将通过短信验证码验证您的身份；登录时需提供手机号及验证码（或密码）。如您不提供，将无法完成注册或登录。

非必要信息：您可以选择绑定邮箱地址、第三方账号（如微信、Apple 账号），用于提升账号安全性和提供更多登录方式。拒绝提供不影响注册与登录功能，但会降低账号安全保护能力。

为防范账号风险，我们会同步收集您的设备信息（设备型号、操作系统、Android ID / IDFV）用于身份验证和安全防护。

（二）跑步功能

必要信息（户外跑）：使用马拉松跑、公路跑、越野跑等户外模式时，我们需获取您的精确位置（GPS），用于实时定位和轨迹记录。如不授权，将无法使用户外跑功能，但不影响室内跑。

运动传感器（非必要）：为提升数据精度，我们可能读取以下传感器数据：

  • 加速度传感器——计算步数、步频
  • 气压传感器——计算海拔和爬升数据
  • 陀螺仪/重力传感器——设备姿态检测、辅助定位
  • 磁力传感器——判断运动数据合理性

如不提供，不影响基础跑步功能，但部分数据精度可能下降。

健康/运动权限（非必要）：iOS 端通过 HealthKit 获取/写入心率、步数等数据；Android 端通过系统运动健身权限获取相应数据。您可随时在系统设置中撤回授权。

存储/相册权限（非必要）：跑步完成后，如需保存动态轨迹图至相册，需要相册/存储权限。如不授权，仅无法保存图片，不影响跑步记录本身。

蓝牙权限（非必要）：如需连接外部心率带、智能手表等设备，需要蓝牙权限。如不授权，仅无法使用外部设备连接功能。

（三）消息推送功能

必要信息：为向您发送系统消息（运动提醒、账号通知等），我们会收集和使用您的设备推送标识符（APNs Token / FCM Token 及各厂商推送 Token），以及网络信息和设备信息，并将其共享至推送 SDK（详见第四条第 4.6 节）。

您可以随时在设备系统设置中关闭通知权限，关闭后将无法收到任何推送消息。

（四）个人信息编辑

您可以在个人主页编辑头像、昵称等信息，上述信息属于非必要信息，如不提供将显示系统默认值。

更换头像时，如从相册选取图片，需要相册权限；如拍摄新照片，需要相机权限。上述权限均为非必要权限，如不授权，您仍可使用应用内其他功能。

（五）图片保存与分享

图片保存：保存跑步战报、轨迹图等内容至本地相册，需要相册/存储权限。如不授权，可以通过截屏方式保存，不影响其他功能。

内容分享：向外部 App（微信、微博等）分享跑步数据时，我们会收集设备信息和操作系统信息，并共享至分享 SDK。Android 端为提供智能分享推荐，可能获取已安装应用列表（非必要，可拒绝授权，仍可手动选择分享目标）。

（六）健康档案与 AI 运动分析功能

为向您提供个性化训练建议和 AI 运动档案分析，我们可能收集：

  • 年龄、性别、身高、体重（非必要，可跳过）
  • 最大心率、静息心率、乳酸阈心率、最大摄氧量（VO2Max）（非必要，可手动填写或由设备同步）
  • HRV（心率变异性）、睡眠时长、心率恢复等生理数据（非必要，通过连接设备获取）
  • 历史跑量和训练负荷数据（由您的跑步记录自动生成）

以上信息均为非必要信息，您可以选择不填写。如不提供，将不影响基础跑步功能，但无法获得基于个人档案的个性化分析服务。我们承诺不将您的健康档案信息用于本政策未声明的其他用途，也不向第三方提供您的个人档案数据。

（七）风控与安全管理

为保障您的账号安全和服务安全运行，防范欺诈和恶意操作，我们会主动收集以下信息用于安全风控：

  • 设备标识符：Android ID、OAID（可选 IMEI/IMSI，需用户授权）；iOS：IDFV
  • 设备基本信息：设备型号、系统版本、应用版本
  • 运行状态信息：进程信息、崩溃日志、应用性能数据
  • 网络信息：IP 地址、运营商名称、接入网络类型
  • Wi-Fi 信息（非必要）：Wi-Fi SSID / BSSID，用于网络环境检测和位置辅助定位

在您进行会员充值时，为完成交易安全校验，我们还会将支付金额、支付时间、设备信息共享至第三方支付 SDK（详见第四条第 4.6 节）。

第三条 个人信息的存储

3.1 存储地点

您的个人信息主要存储在中华人民共和国境内的服务器上。目前，我们不会将您的个人信息传输至境外。如因业务需要确需向境外传输的，我们将严格按照《个人信息保护法》等法律法规的要求，履行安全评估或认证等合法性手续，并获得您的单独同意。

3.2 存储期限

  • 账户信息：账户注销后 30 天内删除或匿名化处理
  • 运动健康数据：账户注销后立即删除，或根据您的要求随时删除
  • 日志数据：保存不超过 12 个月，用于安全分析和问题排查
  • 交易记录：根据《电子商务法》等相关法律法规要求，保存不少于 3 年
  • 超出上述期限后，我们将依法删除您的个人信息或进行匿名化处理

3.3 本地存储

部分数据（如缓存数据、用户设置、本地跑步记录）会存储在您的设备本地，这些数据在您卸载应用时会被自动删除。

3.4 服务停止运营处理

如我们的产品或服务发生停止运营的情形，我们将：

  • 提前以推送通知、应用内公告或其他合理方式及时通知您；
  • 在停止运营后合理期限内，停止继续收集您个人信息的活动；
  • 对所持有的您的个人信息进行删除或匿名化处理，法律法规另有要求的除外。

第四条 个人信息的共享、转让和公开披露

4.1 共享

我们不会与第三方共享您的个人信息，但以下情况除外：

  • 获得您的明确同意；
  • 与我们的关联公司共享，用于提供一致的服务体验，关联公司如需改变个人信息处理目的，将重新征得您的授权；
  • 与授权合作伙伴共享，但仅限于实现本政策声明目的所必需的范围，合作伙伴不得将信息用于其他任何目的；
  • 根据法律法规要求或政府主管部门的合法要求；
  • 您主动选择分享（如向外部平台分享跑步动态）。

4.2 转让

除非获得您的明确同意，我们不会将您的个人信息转让给任何第三方。在合并、收购、分立、解散或破产清算等情况下，如涉及个人信息转让，我们会告知您接收方名称及联系方式，并要求新持有您个人信息的公司继续受本政策约束；若接收方拟变更处理目的或方式，将重新向您征求明示同意。

4.3 公开披露

原则上，我们不会公开披露您的个人信息。下述情况除外：

  • 获得您的明确同意；
  • 公布账号中奖或处罚通知时展示必要的脱敏信息；
  • 基于法律规定或合理怀疑违反法律的情况下。

4.4 共享、转让、公开披露个人信息时无需事先征得授权同意的例外情形

根据《个人信息保护法》及相关法律法规，以下情形中，共享、转让、公开披露您的个人信息无需事先征得您的授权同意：

  • 与国家安全、国防安全有关的；
  • 与公共安全、公共卫生、重大公共利益有关的；

（四）小米运动健康 / vivo 健康 / 其他品牌平台

在您授权后，我们获取：步数、距离、海拔、热量、锻炼记录概要、锻炼详情及位置数据。

（五）Strava

当您授权连接 Strava 账户时，我们从 Strava 获取您的运动活动数据，包括但不限于：活动记录、运动轨迹、距离、时间、配速、心率等。我们对 Strava 数据的处理承诺如下：

  • 数据使用限制：仅将 Strava 数据用于为您提供运动数据同步和分析服务，不向任何第三方披露；
  • 用户隐私保护：即使某些数据在 Strava 平台上设置为公开，我们也不会在本 App 中向其他用户展示您的 Strava 数据；
  • 数据安全：采用业界标准的安全措施保护您的 Strava 数据，确保传输和存储安全；
  • 授权管理：您可随时在 App 内撤销对 Strava 的授权，撤销后我们将停止获取新数据；
  • 合规要求：严格遵守 Strava API 协议，不创建与 Strava 竞争或复制其功能的应用程序。

如果您不授权上述任何第三方数据同步，将无法使用对应的数据同步功能，但不影响 App 内其他功能的正常使用。

第五条 个人信息的保护

5.1 安全措施

  • 数据加密：采用 SSL/TLS 协议加密数据传输；对存储中的敏感个人信息采用加密保存
  • 访问控制：严格限制对个人信息的访问权限，仅授权必要人员访问；部署最小权限原则
  • 安全审计：定期进行安全评估、漏洞扫描和个人信息安全影响评估
  • 员工培训：对处理个人信息的员工进行安全和隐私保护培训
  • 应急响应：建立完善的安全事件应急响应机制和预警机制
  • 数据分类分级：建立数据分类分级制度，对不同类别的个人信息采取不同级别的保护措施

5.2 数据最小化原则

我们遵循数据最小化原则，只收集实现服务功能所必需的最少信息，并在达到目的后及时删除不必要的数据。

5.3 安全事件处理

如发生个人信息安全事件，我们会：

  • 立即启动应急响应机制，采取措施阻止安全事件扩大；
  • 及时评估事件影响并采取补救措施；
  • 按照法律要求及时向监管部门报告；
  • 以邮件、短信、应用内通知、推送等方式及时告知受影响的用户安全事件基本情况、可能影响、已采取措施及您可自主防范和降低风险的建议；
  • 如难以逐一通知，将以合理、有效的方式发布公告。

我们将不定期更新并公开安全风险和个人信息安全影响评估报告的相关内容，您可通过应用内公告获取。
