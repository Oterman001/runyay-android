## 一、软件概述

### 1.1 软件简介

跑步运动助手是一款面向跑步爱好者的 Android 智能运动管理软件。软件基于 Jetpack Compose + Material 3 设计规范开发，采用 Clean Architecture + MVVM 架构模式，为用户提供跑步数据采集、同步、分析、展示及社交分享等全流程服务。

软件支持与 Garmin（国内版/国际版）、COROS（高驰）等主流运动手表品牌的数据对接，能够自动同步用户跑步记录，并提供丰富的运动数据分析能力，帮助用户科学记录跑步历程、追踪训练进展、提升跑步表现。

### 1.2 目标用户

- 拥有 Garmin 或 COROS 运动手表的跑步爱好者
- 希望系统化管理跑步训练数据的用户
- 有长期训练计划、关注运动表现提升的跑步爱好者

### 1.3 核心功能概览

| 功能域 | 核心能力 |
|--------|----------|
| 用户认证 | 注册、登录、找回密码、账号管理 |
| 数据源对接 | Garmin/COROS OAuth 授权、自动数据同步 |
| 跑步记录 | 记录列表、详情展示、地图轨迹可视化 |
| 数据分析 | 周/月/年/总统计、运动目标追踪、心率分区分析 |
| 跑鞋管理 | 跑鞋档案、里程追踪、跑鞋与记录关联 |
| 运动分享 | 短图/长图多样式分享卡片生成 |

## 二、系统架构概述

### 2.1 前端架构

软件采用 Android Jetpack Compose 声明式 UI 框架，基于 Material 3 组件库构建用户界面。架构模式遵循 Clean Architecture 分层设计：

- **Presentation 层**：各功能模块的 Screen（Composable 界面）与 ViewModel，通过 StateFlow 驱动 UI 状态。
- **Domain 层**：纯业务模型（UserInfo、RunRecord、RunSegment、RunningShoe 等），不依赖任何框架。
- **Data 层**：Repository 实现类，整合本地数据库（Room）与远程网络（Retrofit）数据源。

页面导航使用 Jetpack Navigation Compose，路由定义统一维护于 `Screen` 密封类中；部分需要独立生命周期的功能（跑步详情、运动分享）以独立 Activity 承载。

### 2.2 本地数据存储

| 存储方式 | 用途 |
|----------|------|
| Room 数据库（run_database） | 跑步记录、轨迹点、分段数据、个人最佳、VDOT 记录、跑鞋档案、日健康数据等结构化数据 |
| SharedPreferences（user_prefs） | 用户登录态（Token）、个人偏好设置（主题、目标、心率区间参数、看板卡片配置等） |
| SharedPreferences（data_source_prefs） | 各数据源平台绑定状态、上次同步时间、数据源排序等 |

Room 数据库包含以下主要数据表：

- `run_record`：跑步记录主表，存储每次跑步的起止时间、距离、配速、心率、功率、步频、步幅、爬升、卡路里、跑力（VDOT）、训练效果与负荷、数据来源、关联跑鞋 ID、跑步笔记等。
- `run_sample_point`：逐秒采样点数据，存储 GPS 坐标、心率、功率、配速、步频、步幅、垂直振幅、触地时间、累计距离等时序数据。
- `run_segment`：分段数据，包含公里分段（每公里配速/心率/功率汇总）与训练分段（按结构化训练步骤分段）。
- `running_shoe`：跑鞋档案，存储品牌、型号、尺码、颜色、累计里程、预期寿命、状态等。
- `overall_vdot`：整体跑力历史趋势记录。
- `daily_health`：日健康数据（静息心率、最大摄氧量等）。
- `pb_record`：各赛事距离个人最佳记录。

### 2.3 网络通信

软件通过 Retrofit + OkHttp 与服务端进行 HTTPS 通信，统一使用 JSON 格式传输数据。接口请求使用统一封装的 `BaseRequest`（含请求头与业务体），响应使用 `BaseResponse<T>`（含状态码、消息与数据体），`code == "0000"` 表示成功。网络请求在 IO 线程执行，通过 Kotlin Coroutines 与 ViewModel 层协作。

## 三、功能模块详细说明

### 3.1 用户认证模块

用户认证模块提供软件的账号体系管理能力，是所有个性化功能的基础入口。

#### 3.1.1 欢迎页

软件启动后展示欢迎页，提供两个入口按钮供用户选择：进入登录流程或进入注册流程。欢迎页为未登录用户的唯一入口，已登录用户启动后直接进入主页。

#### 3.1.2 登录功能

用户通过手机号与密码进行登录。登录界面提供以下能力：

- **表单校验**：手机号格式合法性验证、密码非空验证，校验不通过时在输入框下方展示错误提示。
- **协议勾选**：用户须勾选同意《用户服务协议》与《隐私政策》后方可登录，协议文本支持点击跳转查看。
- **登录失败保护**：连续登录失败时，系统记录剩余尝试次数并在界面提示。当尝试次数耗尽时，系统提示用户通过"忘记密码"流程重置账号。
- **辅助入口**：登录页提供"忘记密码"、"联系我们"、"注册账号"的快捷跳转入口。
- **登录成功流转**：登录成功后，系统检查该账号是否已完成生理信息录入；未完成者引导至生理信息设置页，已完成者直接进入主页。

#### 3.1.3 注册功能

新用户注册采用多步骤流程，各步骤依次完成：

1. **手机号输入步骤**：输入手机号并同意协议，点击获取验证码，系统向该手机号发送短信验证码（业务场景类型：REGISTER）。
2. **验证码验证步骤**：输入收到的短信验证码完成手机号验证；支持倒计时结束后重新发送验证码。
3. **密码设置步骤**：设置账号登录密码，密码须满足复杂度要求（长度、字符类型等）；完成后提交注册请求，服务端创建账号并返回登录 Token。
4. **注册完成**：注册成功后自动进入生理信息设置引导流程。

#### 3.1.4 找回密码功能

忘记密码同样采用多步骤流程：

1. **手机号输入步骤**：输入已注册的手机号，请求发送重置验证码（业务场景类型：RESET）。
2. **验证码验证步骤**：输入短信验证码，服务端验证通过后授权重置。
3. **新密码设置步骤**：设置并确认新密码，提交成功后返回登录页，提示用户使用新密码登录。

#### 3.1.5 协议与隐私政策

提供独立的用户服务协议（UserTermsScreen）与隐私政策（PrivacyPolicyScreen）页面，内容通过 WebView 加载展示，支持内容滚动阅读。

### 3.2 新用户引导模块

新用户完成注册或登录后（服务端检测到尚无生理信息记录），系统自动引导用户完成基础配置，以便后续精准计算心率分区、跑力等运动指标。

#### 3.2.1 生理信息设置

用户在此页面录入以下基础生理参数：

| 参数 | 说明 |
|------|------|
| 最大心率（bpm） | 用于计算心率训练区间，可手动输入或由系统根据年龄推算 |
| 静息心率（bpm） | 反映基础有氧能力，参与储备心率区间计算 |
| 出生日期 | 参与最大心率推算（220 减年龄公式参考值）及年龄相关分析 |
| 性别 | 参与生理指标基准计算 |

信息提交后保存至服务端（`/api/user/basic-info/save`），同时在本地持久化缓存。引导完成后跳转至数据源绑定引导页。

#### 3.2.2 数据源绑定引导

引导用户绑定其所持有的运动手表数据源（Garmin 或 COROS），以便软件自动同步历史和最新跑步记录。用户可选择跳过该步骤，后续在数据源管理功能中随时绑定。完成或跳过后进入软件主页。

### 3.3 主页与看板模块

主页是软件的核心容器，采用底部导航栏设计，包含多个标签页（Tab），满足用户日常使用的高频功能入口需求。

#### 3.3.1 看板（Dashboard）标签页

看板以卡片形式汇总展示用户的跑步运动概览：

- **跑步目标进度卡片**：根据用户设定的年度或月度里程/时长目标，展示当前完成进度与剩余目标。
- **近期统计卡片**：展示最近一段时间的跑步次数、总里程、平均配速等汇总数据。
- **个人最佳（PB）卡片**：展示用户在各赛事距离（5K、10K、半马、全马等）上的个人最佳成绩。
- **跑力（VDOT）卡片**：展示用户的综合跑步能力指数，反映有氧运动能力水平。
- **轨迹墙卡片**：以缩略图形式展示近期跑步轨迹，形成可视化的运动记录墙。
- **热力图卡片**：以热力图形式呈现用户跑步频率与里程的时间分布。

看板卡片支持用户自定义可见性，用户可根据个人偏好选择展示或隐藏各卡片。

#### 3.3.2 数据（Data）标签页

数据标签页以列表形式展示用户的所有跑步记录，支持按时间倒序浏览。每条记录展示摘要信息（日期、距离、配速、时长、心率等）。点击记录条目进入跑步记录详情页。

#### 3.3.3 个人中心（Profile）标签页

个人中心提供账号相关的设置入口，包括：个人资料编辑、跑步目标设置、心率区间设置、跑鞋管理、数据源管理、数据统计、联系我们等。

### 3.4 跑步记录详情模块

跑步记录详情模块以独立页面（Activity）形式展示单次跑步的完整数据，支持地图轨迹、运动指标、分段分析等多维度展示。

#### 3.4.1 地图轨迹展示

- 在地图上绘制本次跑步的 GPS 运动轨迹，支持缩放与拖动。
- **隐私模式**：用户可开启隐私模式，此时轨迹首尾两端的部分路段将被隐藏，保护用户起终点隐私。
- 户外跑（有 GPS 数据）展示地图轨迹；室内跑则展示非地图形式的运动概览。

#### 3.4.2 运动指标展示

以图文形式展示本次跑步的全量运动指标：

| 指标类别 | 指标项 |
|----------|--------|
| 基础指标 | 总里程（km）、运动时长、平均配速（min/km）、最快配速 |
| 心率指标 | 平均心率（bpm）、最大心率（bpm） |
| 步态指标 | 平均步频（步/分）、平均步幅（cm）、垂直振幅（cm）、垂直步幅比（%）、触地时间（ms） |
| 能量指标 | 卡路里消耗（kcal） |
| 功率指标 | 平均功率（W）、最大功率（W）（如设备支持） |
| 综合评估 | 跑力（VDOT）、训练效果、训练负荷、爬升高度（m） |

各指标均配有对应的历史趋势折线图，支持按心率、配速、步频、功率、步幅、垂直振幅、触地时间、海拔等维度查看随时间变化的时序图表。

#### 3.4.3 心率分区分析

根据用户的最大心率与静息心率，将本次跑步心率数据按照训练区间进行分析：

- 支持 **5 区模式**（恢复/有氧/节奏/乳酸阈值/无氧）与 **7 区模式**（更细粒度区间划分）切换。
- 以环形图或柱状图形式展示各心率区间的时间占比。
- 每个区间标注对应的心率范围与训练意义。

#### 3.4.4 分段数据

- **公里分段**：按每公里展示配速、心率、功率等指标，帮助用户分析配速稳定性与体能分配。
- **训练分段**：针对结构化训练课（如间歇训练），按训练步骤（热身/正式/恢复/冷身等）展示各段数据，支持合并展示重复组别。

#### 3.4.5 笔记编辑

用户可为每次跑步添加文字笔记，记录当次的训练感受、天气状况等个性化备注。笔记内容同步保存至服务端。

#### 3.4.6 跑鞋关联

用户可将本次跑步记录与跑鞋档案关联，系统自动将该次跑步里程计入对应跑鞋的累计里程。已关联跑鞋在详情页展示跑鞋品牌型号信息。

### 3.5 运动数据统计模块

运动数据统计模块提供多时间维度的跑步数据聚合分析，帮助用户从宏观视角了解运动进展。

#### 3.5.1 统计维度

软件提供以下四个时间维度的统计标签页，可通过顶部 Tab 切换：

| 维度 | 统计范围 | 核心指标 |
|------|----------|----------|
| 周统计 | 当周（周一至周日） | 周里程、运动次数、周均配速、周最长跑 |
| 月统计 | 当月（自然月） | 月里程、运动次数、月均配速、月最长跑 |
| 年统计 | 当年（自然年） | 年里程、运动次数、月均里程趋势图 |
| 全部统计 | 账号全量历史数据 | 总里程、总次数、最长单次跑、年度里程趋势 |

各维度统计页均展示所选时段内的运动次数、总里程、总时长、平均配速、平均心率、总爬升等核心汇总指标，并配有相应的可视化图表。

#### 3.5.2 跑步目标设置

用户可设置个性化的跑步训练目标：

- **目标类型**：里程目标（km）或时长目标（小时）。
- **目标周期**：年度目标或月度目标。
- **目标启用开关**：可随时开启或关闭目标追踪功能。

目标设置保存于本地偏好存储，看板页的目标进度卡片将实时反映当前完成进度。

### 3.6 数据源管理模块

数据源管理模块负责管理与第三方运动设备平台的数据对接关系，实现跑步数据的自动同步。

#### 3.6.1 支持的数据源平台

| 平台 | 说明 |
|------|------|
| Garmin Connect（国内版） | 适用于佳明国行设备用户（garmin.cn 账号体系） |
| Garmin Connect（国际版） | 适用于佳明国际版设备用户（connect.garmin.com 账号体系） |
| COROS（高驰） | 适用于高驰运动手表用户 |

#### 3.6.2 数据源绑定（OAuth 授权）

用户选择相应平台后，软件发起 OAuth 授权流程：

1. 软件向服务端请求授权跳转 URL，服务端返回对应平台的 OAuth 授权页地址。
2. 软件内嵌 WebView 加载授权页，用户在该页面完成第三方平台账号的登录与授权。
3. 授权完成后，WebView 接收回调参数（OAuth Token 与 Verifier），转发至服务端完成绑定握手。
4. 绑定成功后，数据源列表中该平台标记为"已绑定"状态。

#### 3.6.3 数据自动同步

绑定成功后，软件支持数据自动同步：

- 用户可手动触发同步，软件从对应平台拉取自上次同步时间起的增量跑步记录。
- 支持查看同步进度与同步状态（同步中/同步成功/同步失败）。
- 同步数据包括：跑步基础摘要、GPS 轨迹点、分段数据及其他设备采集指标。

#### 3.6.4 数据源解绑

用户可随时解除与某平台的绑定关系。解绑操作调用服务端解绑接口，完成后该平台恢复为"未绑定"状态，已同步至本地的历史数据不受影响。

### 3.7 FIT 文件手动导入模块

FIT 文件手动导入模块为用户提供主动导入跑步数据的能力，无需依赖第三方平台的 API 同步，适用于从设备本地读取数据或从其他设备导出 FIT 格式文件的场景。

#### 3.7.1 文件选择与上传

- 用户通过系统文件选择器选择设备存储中的一个或多个 FIT 格式文件（支持多选批量导入）。
- 系统对所选文件进行格式校验，仅接受符合 ANT/Garmin FIT 规范的文件。
- 文件上传至服务端（`/api/fit/upload`），服务端解析 FIT 文件内容并生成跑步记录摘要，返回上传状态与解析结果。
- 上传时携带设备类型、活动类型、开始时间等元数据，便于服务端准确识别记录来源。

#### 3.7.2 已导入记录列表

导入完成后，用户可在已导入列表中查看通过手动导入方式同步的跑步记录，支持点击跳转至对应记录的平台详情页或跑步记录详情页。

### 3.8 跑鞋管理模块

跑鞋管理模块为用户提供专业的跑步装备档案管理能力，通过里程追踪与寿命预警，帮助用户科学管理跑鞋的使用周期。

#### 3.8.1 跑鞋列表

以卡片列表形式展示用户的所有跑鞋档案，分"在用"与"已退役"两个状态区分展示。每张卡片展示跑鞋图片（缩略图）、品牌型号、累计里程与预期寿命进度条。

#### 3.8.2 添加与编辑跑鞋

用户可录入以下跑鞋信息：

| 字段 | 说明 |
|------|------|
| 品牌（Brand） | 如 Nike、Adidas、ASICS 等 |
| 型号（Model） | 具体鞋款名称 |
| 鞋码（Size） | 鞋子尺码 |
| 昵称 | 用户自定义鞋子别名 |
| 类型（Type） | 训练鞋/比赛鞋/越野鞋等 |
| 颜色 | 跑鞋颜色 |
| 价格 | 购入价格（选填） |
| 预期寿命（km） | 期望跑鞋使用到的最大里程数 |
| 首次使用日期 | 跑鞋开始使用的日期 |
| 初始里程（km） | 跑鞋在此软件中开始追踪前已跑里程（支持从中途开始记录） |
| 备注 | 自定义说明文字 |
| 跑鞋图片 | 支持从相册选取或拍照上传 |

跑鞋信息保存至服务端并在本地缓存，编辑时支持修改所有字段。

#### 3.8.3 里程追踪与退役管理

- 每次跑步记录与跑鞋关联后，系统自动将当次里程累加至该跑鞋的累计里程。
- 当累计里程接近预期寿命时，系统在列表页以进度条颜色变化提示用户关注跑鞋状态。
- 用户可手动将跑鞋标记为"已退役"，退役后的跑鞋不再出现在记录关联的选择列表中，但历史数据完整保留。

#### 3.8.4 跑鞋与记录关联

- **单条关联**：在跑步记录详情页选择关联的跑鞋。
- **批量关联**：在跑鞋详情页，通过批量关联功能一次性为多条历史跑步记录指定同一双跑鞋。
- 跑鞋详情页展示关联该跑鞋的所有跑步记录列表，支持点击跳转至记录详情。

### 3.9 心率区间设置模块

心率区间设置模块允许用户精细化配置个人心率训练区间参数，确保心率分析与训练区间计算的准确性。

#### 3.9.1 心率参数设置

| 参数 | 说明 |
|------|------|
| 最大心率（HRmax） | 用于计算百分比区间；可手动精确输入，覆盖系统推算值 |
| 静息心率（HRrest） | 用于计算储备心率（Karvonen 公式）区间 |

#### 3.9.2 心率区间模式

- **5 区模式**：基于最大心率百分比划分 5 个训练区间（恢复区/有氧基础区/节奏区/乳酸阈值区/无氧区），适合大多数跑步训练场景。
- **7 区模式**：更细粒度的 7 区划分，适合有系统化训练计划需求的进阶用户。

#### 3.9.3 与平台同步

用户可选择是否将心率区间参数同步至 Garmin 或 COROS 平台，实现与运动手表的心率区间设置保持一致。

### 3.10 跑步分享模块

跑步分享模块为用户提供将跑步成绩可视化分享至社交媒体的能力，支持多种样式的分享图片生成。

#### 3.10.1 分享模式

软件提供两种分享图片样式：

- **短图模式（SHORT）**：生成一张简洁的运动成绩卡片，用户可从可用指标中自由选择展示的核心数据项（如配速、里程、时长、心率、卡路里等），适合快速社交分享。
- **长图模式（LONG）**：生成包含丰富数据的详细分享长图，以卡片组合形式展示地图轨迹、核心指标、心率/配速趋势图、心率区间分析、公里分段等完整数据，适合记录展示需求。

#### 3.10.2 隐私保护

分享图片生成时支持**隐私模式**，开启后地图轨迹上的起终点区域将被遮盖，保护用户的家庭住址等位置隐私，同时保留跑步路线的整体形态。

#### 3.10.3 内容个性化

用户可在分享前自定义以下内容：

- **展示/隐藏日期**：控制是否在分享图中显示跑步日期。
- **展示/隐藏昵称**：控制是否在分享图中显示用户昵称。
- **设备名称**：自定义展示在分享图中的运动设备名称文字。
- **品牌文案**：随机生成的个性化品牌标语（可随机刷新）。
- **心率区间模式**：支持在短图/长图中切换使用 5 区或 7 区心率展示模式。

用户自定义的分享偏好（指标选择、卡片开关、日期/昵称显示偏好等）在本地持久化存储，下次分享时自动恢复上次设置。

#### 3.10.4 导出方式

- **保存到相册**：将生成的图片以 JPG 格式保存至设备相册，文件名格式为 `run_share_{记录ID}_{时间戳}_{short|long}.jpg`。
- **系统分享**：调用 Android 系统分享面板，支持分享至微信、微博、抖音等任意已安装的社交应用。

### 3.11 个人资料与账号管理模块

个人资料与账号管理模块提供用户账号信息的维护入口及客服支持渠道。

#### 3.11.1 个人资料编辑

- **头像修改**：支持从手机相册选取图片作为头像，图片经裁剪后上传至服务端（`/api/user/profile/avatar`），并在本地缓存以加速加载。
- **昵称修改**：用户可修改账号昵称，更新后同步至服务端（`/api/user/profile/nickname`）并在主页、分享图片等处同步刷新显示。

#### 3.11.2 联系我们

提供联系方式页面，展示软件运营方的联系渠道，包含微信公众号二维码展示页，用户可扫码关注公众号获取服务支持与产品动态。

#### 3.11.3 账号登出

用户可在个人资料页安全退出当前登录账号。登出操作清除本地存储的 Token 与用户信息，并调用服务端登出接口（`/api/user/login/logout`），完成后跳转至欢迎页。

#### 3.11.4 账号注销

用户可申请永久注销当前账号。注销操作需要用户输入当前密码进行身份验证，并填写注销原因，经服务端确认后（`/api/user/login/deactivate`）完成注销，账号数据依据隐私政策进行处理。

## 四、数据安全与隐私说明

### 4.1 用户认证安全

- 密码在网络传输过程中经过加密处理，服务端以密文存储，不以明文保存用户密码。
- 登录 Token 带有有效期，软件每日自动刷新 Token（`/api/user/login/refreshToken`），Token 过期后自动跳转至登录页要求重新认证。
- 登录失败次数超过限制时，系统锁定密码登录入口，引导用户通过手机验证码重置密码，防止暴力破解。

### 4.2 位置隐私保护

- GPS 轨迹数据仅用于跑步记录的展示与分析，不上传至第三方广告或商业平台。
- 提供隐私模式功能，用户可选择在地图展示及分享图片中隐藏轨迹首尾区域，避免暴露家庭或工作地点。

### 4.3 本地数据存储安全

- Room 数据库文件存储于应用私有存储目录，无 root 权限的情况下无法被其他应用访问。
- SharedPreferences 中的用户 Token 等敏感信息存储于应用私有沙盒目录。

### 4.4 权限声明

软件申请以下 Android 系统权限，均有明确功能对应，不申请超出功能范围的权限：

| 权限 | 用途 |
|------|------|
| ACCESS_FINE_LOCATION | FIT 文件详情中地图轨迹展示 |
| INTERNET | 与服务端进行数据通信 |
| READ_EXTERNAL_STORAGE / READ_MEDIA_IMAGES | 手动导入 FIT 文件、选取头像/跑鞋图片 |
| WRITE_EXTERNAL_STORAGE | 分享图片保存至相册 |
| POST_NOTIFICATIONS | 数据同步进度通知推送 |
| CAMERA | 拍照上传跑鞋图片（可选） |

## 五、附录：核心数据流程

### 5.1 数据同步流程

```
用户触发同步
    │
    ▼
服务端从 Garmin/COROS 拉取增量活动文件列表
    │
    ▼
按平台逐文件下载 FIT 数据并解析
    │
    ▼
解析结果写入服务端跑步摘要库
    │
    ▼
客户端查询服务端摘要列表（分页）
    │
    ▼
写入本地 Room 数据库（run_record 表）
    │
    ▼
按需下载逐秒采样点数据（GPS 轨迹 + 心率等）
    │
    ▼
主页记录列表刷新展示
```

### 5.2 跑步记录展示流程

```
用户点击记录条目
    │
    ▼
RunDetailActivity 启动，传入 workoutId
    │
    ▼
RunDetailViewModel 并行加载：
    ├── 本地 Room 查询跑步摘要（run_record）
    ├── 本地 Room 查询 GPS 采样点（run_sample_point）
    ├── 本地 Room 查询分段数据（run_segment）
    └── 服务端查询跑力 / 日健康数据
    │
    ▼
地图轨迹渲染（Mapbox）
    │
    ▼
指标卡片、图表、心率区间分析渲染
    │
    ▼
用户可操作：编辑笔记 / 关联跑鞋 / 进入分享页
```
