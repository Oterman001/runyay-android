## 一、软件概述

### 1.1 软件简介

跑步运动助手是一款面向跑步爱好者的 Android 智能运动管理软件，基于 Jetpack Compose + Material 3 设计规范开发，采用 Clean Architecture + MVVM 架构模式，为用户提供跑步数据采集、同步、分析、展示及社交分享等全流程服务。

软件支持与 Garmin（国内版/国际版）、COROS（高驰）等主流运动手表品牌的数据对接，能够自动同步用户跑步记录，并提供丰富的运动数据分析能力，帮助用户科学记录跑步历程、追踪训练进展、提升跑步表现。

### 1.2 主要功能模块

| 功能模块 | 核心能力 |
|----------|----------|
| 账号管理 | 注册、登录、找回密码、账号资料管理 |
| 数据源对接 | Garmin/COROS OAuth 授权、数据自动同步 |
| 跑步记录 | 记录列表、详情展示、地图轨迹可视化 |
| 数据分析 | 周/月/年/总统计、运动目标追踪、心率分区分析 |
| 跑鞋管理 | 跑鞋档案管理、里程追踪、记录关联 |
| 运动分享 | 短图/长图多样式分享卡片生成 |

### 1.3 软件主要特色

- **多平台数据同步**：支持 Garmin、COROS 等主流运动手表一键授权，自动拉取历史与实时跑步数据，无需手动导入。
- **专业数据分析**：提供心率分区（5 区/7 区）、公里分段、训练分段、跑力（VDOT）趋势等专业运动指标分析。
- **跑鞋里程追踪**：建立跑鞋档案，自动累计每次跑步里程，智能提示跑鞋更换时机。
- **个性化运动分享**：生成精美短图/长图分享卡片，支持隐私模式遮盖轨迹起终点，一键分享至社交媒体。
- **隐私保护设计**：所有 GPS 数据存储于设备本地沙盒，地图及分享图均支持隐私模式，保护用户位置信息。

## 三、账号管理

账号管理模块是软件的入口，负责用户的注册、登录及找回密码功能。软件首次启动或用户未登录时，自动跳转至欢迎页。

### 3.1 欢迎页

**进入方式：** 软件首次安装后启动，或登出/注销账号后自动跳转至本页面。

欢迎页是软件的起始界面，以品牌形象为主视觉，提供两个操作入口：

- **登录**：已有账号的用户点击进入登录流程。
- **注册**：新用户点击进入注册流程。

已登录用户启动软件后将直接跳过欢迎页，进入主页。

### 3.2 用户登录

**进入方式：** 在欢迎页点击「登录」按钮，进入登录页面。

用户通过手机号与密码进行登录。登录页面提供以下功能：

- **手机号输入**：输入已注册的手机号（11 位数字），系统自动校验格式。
- **密码输入**：输入账号密码，支持显示/隐藏密码切换。
- **协议勾选**：用户须勾选同意《用户服务协议》与《隐私政策》方可登录；协议文本支持点击跳转查看。
- **登录失败保护**：连续登录失败时，界面实时提示剩余尝试次数；尝试次数耗尽后自动禁止密码登录，提示用户通过"忘记密码"流程重置。
- **辅助入口**：页面底部提供"忘记密码"、"注册账号"快捷入口，右侧提供"联系我们"入口。

**登录成功后的跳转逻辑：**

```
登录成功
  ├── 服务端返回：该账号尚未填写生理信息 → 跳转至「生理信息设置」页
  └── 服务端返回：生理信息已完成 → 直接跳转至「主页」
```

### 3.3 用户注册

**进入方式：** 在欢迎页点击「注册」按钮，或在登录页点击「注册账号」链接，进入注册流程。

注册采用三步骤引导流程，各步骤通过顶部进度条展示当前所处阶段。

#### 第一步：手机号输入

用户输入手机号并勾选同意协议，点击「获取验证码」，系统向该号码发送 6 位数字短信验证码（有效期 5 分钟）。验证码发送成功后，自动进入第二步。

#### 第二步：验证码验证

用户输入收到的短信验证码（6 位数字），系统实时验证。验证码输入框支持以下操作：

- 倒计时结束前，"重新发送"按钮显示为灰色并倒计时；倒计时结束后可点击重新发送。
- 验证码正确后自动进入第三步。

#### 第三步：密码设置

用户设置账号登录密码，密码须满足复杂度要求（8 位以上，包含字母与数字）。点击「完成注册」提交后：

- 服务端创建账号，返回登录 Token。
- 注册成功后自动登录，跳转至「生理信息设置」引导页。

**进入方式：** 在登录页点击「忘记密码」链接，进入找回密码流程。

3. **新密码设置**：输入并确认新密码，提交成功后跳转回登录页，提示用户使用新密码登录。

## 四、新用户初始化引导

新用户完成注册后、或已有账号首次在新设备登录且服务端检测到尚无生理信息记录时，系统自动进入初始化引导流程，帮助用户完成基础配置。

### 4.1 生理信息设置

**进入方式：** 注册完成后自动跳转，或登录成功后服务端检测到生理信息未填写时自动跳转。

用户在此页面录入基础生理参数，这些数据将用于后续心率分区计算与跑力（VDOT）分析：

| 参数 | 说明 | 填写方式 |
|------|------|---------|
| 最大心率（bpm） | 决定心率训练区间上限 | 手动输入，或系统按"220 - 年龄"公式自动推算 |
| 静息心率（bpm） | 反映基础有氧能力，参与储备心率区间计算 | 手动输入 |
| 出生日期 | 用于年龄推算及数据分析 | 日期选择器 |
| 性别 | 参与生理指标基准计算 | 单选 |

填写完成后点击「保存」，数据同步至服务端，随后自动跳转至「数据源绑定引导」页。

![图4-1 生理信息设置页](images/screenshot_07_physio_setup.png)

## 五、主页

**进入方式：** 完成数据源绑定引导或点击「跳过」后进入；后续每次打开软件（已登录状态）直接进入。

主页是软件的核心容器，采用底部导航栏设计，包含三个标签页，用户可通过底部导航栏图标随时切换。

### 5.1 看板（Dashboard）标签页

看板标签页是主页默认显示的标签，以卡片形式汇总展示用户的跑步运动概览数据。

**主要展示卡片（可自定义显示/隐藏）：**

- **跑步目标进度卡片**：根据用户设定的年度或月度里程/时长目标，以进度环形式展示当前完成百分比与剩余目标量。
- **近期统计卡片**：展示最近一段时间内的跑步次数、总里程、平均配速等汇总数据。
- **个人最佳（PB）卡片**：展示用户在 5K、10K、半马、全马等各赛事距离的个人最佳成绩。
- **跑力（VDOT）卡片**：展示综合跑步能力指数趋势，反映有氧运动能力的历史变化。
- **轨迹墙卡片**：以缩略地图图形式展示近期跑步轨迹，形成可视化运动记录墙，支持多列展示模式切换。
- **热力图卡片**：以日历热力图形式呈现用户跑步频率与里程随时间的分布规律。

**操作说明：** 下拉可刷新数据；长按卡片可进入自定义排序/隐藏模式。点击各卡片可跳转至对应的详情功能页。

## 六、跑步记录详情

**进入方式：** 在主页「数据」标签页的记录列表中，点击任意一条跑步记录卡片，进入该次跑步的详情页面。

跑步记录详情以独立页面（Activity）承载，完整展示单次跑步的所有数据维度。

### 6.1 地图轨迹展示

详情页顶部为地图区域，在地图上以彩色线段绘制本次跑步的 GPS 运动轨迹，颜色深浅可反映配速变化：

- **缩放与拖动**：支持双指缩放、单指拖动查看轨迹全貌。
- **隐私模式**：页面右上角可开启隐私模式；开启后，轨迹首尾各约 500 米路段被遮盖，保护用户家庭/工作地点隐私，跑步路线整体形态保留不变。
- **室内跑**：若本次跑步无 GPS 数据（如跑步机），地图区域将展示为运动数据摘要卡片，不显示地图。

**操作说明：** 地图区域下方为运动数据展示区，向上滑动可查看更多指标详情。

### 6.2 运动指标展示

| 指标类别 | 具体指标 |

### 6.3 心率分区分析

### 8.3 OAuth 授权流程

**进入方式：** 在数据源详情页（8.2）点击「绑定账号」，软件内嵌 WebView 加载第三方平台授权页。

OAuth 授权流程说明：

1. 软件向服务端请求授权地址，服务端返回对应平台（Garmin/COROS）的 OAuth 授权页 URL。
2. 内嵌 WebView 加载授权页，用户在页面中输入第三方平台账号密码并点击授权确认。
3. 授权完成后，第三方平台回调至软件，WebView 截获回调参数（OAuth Token 与 Verifier）。
4. 软件将回调参数转发至服务端完成绑定握手，授权成功后返回数据源详情页，状态更新为「已绑定」。

**注意事项：** 授权页内的账号密码由第三方平台（Garmin/COROS）直接处理，软件不存储第三方账号凭据。

![图8-3 OAuth 授权 WebView 页面](images/screenshot_20_datasource_oauth.png)

## 九、FIT 文件手动导入

**进入方式：** 在数据源管理列表页（8.1）底部点击「手动导入 FIT 文件」入口，进入手动导入页面。

FIT 文件手动导入功能为用户提供不依赖第三方平台 API 的数据导入方式，适用于以下场景：

- 设备存储中存有 FIT 格式运动文件但尚未同步至 Garmin/COROS 云端。
- 从其他运动软件或设备导出 FIT 文件后手动导入。

**操作步骤：**

1. 点击「选择文件」按钮，调用系统文件选择器。
2. 在文件管理器中选择一个或多个 `.fit` 格式文件（支持多选批量导入）。
3. 确认选择后，软件对文件进行格式校验（仅接受符合 ANT/Garmin FIT 规范的文件）。
4. 校验通过的文件逐一上传至服务端，服务端解析 FIT 文件内容并生成跑步记录摘要。
5. 导入完成后，已成功导入的记录显示在下方「已导入记录」列表中，可点击跳转至对应跑步详情。

**注意事项：** 若上传的 FIT 文件已存在于本地记录中（重复文件），服务端将自动去重，不会产生重复记录。

![图9-1 FIT 文件手动导入页](images/screenshot_21_fit_import.png)

## 十一、心率区间设置

**进入方式：** 在主页个人中心标签页（5.3）点击「心率区间」入口，进入心率区间设置页。

心率区间设置页允许用户精细化配置心率训练区间参数，确保跑步详情中心率分区分析的计算准确性。

**心率参数设置：**

| 参数 | 作用 | 设置方式 |
|------|------|---------|
| 最大心率（HRmax，bpm） | 决定各训练区间的上限心率 | 手动输入精确值，覆盖系统按年龄推算的参考值 |
| 静息心率（HRrest，bpm） | 参与储备心率（Karvonen）公式计算 | 手动输入 |

**心率区间模式：**

- **5 区模式（推荐）**：按最大心率百分比划分 5 个区间（恢复区 50-60% / 有氧基础区 60-70% / 节奏区 70-80% / 乳酸阈值区 80-90% / 无氧区 90-100%），适合大多数日常训练场景。
- **7 区模式**：将 5 区进一步细分为 7 个更精细的区间，适合采用专业训练计划（如 Jack Daniels 训练法）的进阶跑步爱好者。

**平台同步选项：** 用户可勾选「同步至 Garmin/COROS」，将软件中设置的心率参数同步至对应运动手表，使手表端与软件端心率区间定义保持一致。

设置保存后立即生效，所有跑步记录的心率分区图表将基于新参数重新计算展示。

## 十三、个人资料与账号管理

### 13.1 个人资料编辑

**进入方式：** 在主页个人中心标签页（5.3）点击「个人资料」入口，进入个人资料页；或点击头像区域直接进入。

个人资料页支持对账号基础信息进行修改：

**头像修改：**

点击头像图片区域，弹出选项菜单（从相册选取 / 拍照）。选择图片后进入裁剪步骤，确认裁剪后上传至服务端（`/api/user/profile/avatar`），上传成功后头像立即在主页与分享图中同步更新。

**昵称修改：**

点击昵称区域，输入框激活，修改后点击「保存」，新昵称同步至服务端（`/api/user/profile/nickname`），并在主页、分享图片等处实时刷新。

**账号登出：**

点击「登出」按钮，清除本地存储的登录 Token 与用户信息，调用服务端登出接口（`/api/user/login/logout`），完成后跳转至欢迎页。

**账号注销：**

点击「注销账号」按钮，进入身份验证步骤（输入当前密码 + 填写注销原因），服务端确认后永久注销账号，账号数据依据隐私政策处理，操作不可撤销。

## 附录A：权限说明

软件申请以下 Android 系统权限，均与软件功能直接对应：

| 权限 | 用途 |
|------|------|
| ACCESS_FINE_LOCATION | 地图轨迹展示（跑步详情页地图功能） |
| INTERNET | 账号认证与数据同步的网络通信 |
| READ_MEDIA_IMAGES / READ_EXTERNAL_STORAGE | FIT 文件导入、头像与跑鞋图片选取 |
| WRITE_EXTERNAL_STORAGE | 分享图片保存至手机相册 |
| POST_NOTIFICATIONS | 数据同步进度通知推送 |
| CAMERA | 拍照上传跑鞋图片（可选，用户拒绝不影响核心功能） |
