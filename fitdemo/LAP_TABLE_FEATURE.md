# 区间数据表格展示功能说明

## 功能更新

已将区间数据的展示方式从卡片列表改为紧凑的表格形式。

## 表格特性

### 1. 表格结构
- **表头**: 包含列名，使用深色背景突出显示
- **数据行**: 每个区间占一行，显示关键数据
- **边框**: 清晰的单元格边框便于阅读

### 2. 显示的列

| 列名 | 宽度 | 说明 |
|------|------|------|
| 区间 | 50dp | 区间编号 (1, 2, 3...) |
| 距离 | 90dp | 本区间跑步距离 |
| 用时 | 80dp | 总用时 (时:分:秒) |
| 配速 | 90dp | 平均配速 (分'秒"/公里) |
| 心率 | 60dp | 平均心率 (bpm) |
| 步频 | 60dp | 平均步频 (spm) |
| 卡路里 | 70dp | 消耗卡路里 |

### 3. 交互功能
- ✅ **横向滚动**: 表格可以左右滑动，查看所有列
- ✅ **纵向滚动**: 跟随页面滚动，查看所有区间
- ✅ **自适应**: 自动适应不同屏幕尺寸

### 4. 视觉设计
- **表头**: 主题色背景 + 粗体文字
- **数据行**: 浅色背景，易于阅读
- **边框**: 细线边框区分单元格
- **对齐**: 所有文本居中对齐
- **字体**: 表头使用 labelMedium，数据使用 bodySmall

## 使用效果

### 对比原方案

**原方案 (卡片列表)**:
- 优点: 显示信息全面（包括开始时间、最大值等）
- 缺点: 占用空间大，需要大量滚动
- 适合: 详细查看单个区间

**新方案 (表格)**:
- 优点: 紧凑，一目了然，方便对比
- 缺点: 显示信息有限（只显示核心数据）
- 适合: 快速浏览和对比多个区间

### 数据展示策略

表格只展示**最常用的核心数据**:
- 距离 - 最重要的指标
- 用时 - 完成时间
- 配速 - 跑步强度
- 心率 - 身体负荷
- 步频 - 跑步节奏
- 卡路里 - 能量消耗

**未在表格中显示的数据**:
- 开始时间 (不常用)
- 计时时间 (与用时类似)
- 最大速度/心率 (不如平均值重要)
- 上升/下降 (地形数据，不是所有跑步都有)

## 代码实现

### 新增文件
- `LapDataTable.kt` - 表格组件实现

### 核心组件

```kotlin
@Composable
fun LapDataTable(laps: List<LapData>) {
    // 表格标题
    SectionHeader("区间数据")
    
    // 横向可滚动表格
    Row(modifier = Modifier.horizontalScroll(...)) {
        Column {
            TableHeader()  // 表头
            laps.forEach { lap ->
                TableRow(lap)  // 数据行
            }
        }
    }
}
```

### 样式特点
- Material 3 设计规范
- 响应主题色变化
- 支持深色模式
- 文字溢出处理

## 可能的改进方向

如果需要，可以考虑：

1. **可展开的详细信息**
   - 点击行可展开显示完整信息
   - 平衡紧凑性和信息完整性

2. **排序功能**
   - 点击表头按该列排序
   - 例如按配速排序找出最快/最慢的区间

3. **数据高亮**
   - 最快配速用绿色标注
   - 最慢配速用红色标注
   - 帮助快速识别表现

4. **更多列选项**
   - 让用户自定义显示哪些列
   - 根据使用习惯调整

5. **导出功能**
   - 将表格数据导出为CSV
   - 方便进一步分析

## 测试建议

1. **多区间测试**: 使用有5+个区间的FIT文件测试
2. **横向滚动**: 验证表格可以流畅左右滑动
3. **不同屏幕**: 在不同尺寸设备上测试显示效果
4. **缺失数据**: 测试某些字段为空时的显示（显示"-"）

## 使用说明

1. 打开应用
2. 选择包含区间数据的FIT文件
3. 滚动到"区间数据"部分
4. 左右滑动查看所有列
5. 上下滚动查看所有区间

现在的表格展示更适合快速浏览和对比多个区间的表现！
