│ - 综合跑力：简单指数衰减加权，无异常值检测；inclusiveLevel                   │
│ 字段存在但未被使用；使用序号而非实际天数做衰减                               │

│ 一、设计决策：哪些因素应/不应影响 VDOT                                       │
│                                                                              │

│ ├──────────┼────────────┼──────────────────────────────────────────────────┤ │
│ │ 性别     │ 不纳入     │ 同理，相同成绩 = 相同 VDOT，性别已体现在值本身   │ │
│ ├──────────┼────────────┼──────────────────────────────────────────────────┤ │
│ │ 温度     │ 已有，优化 │ 结合湿度计算体感温度                             │ │
│ ├──────────┼────────────┼──────────────────────────────────────────────────┤ │

│ │          │            │ RunRecordEntity.weatherHumidity                  │ │
│ ├──────────┼────────────┼──────────────────────────────────────────────────┤ │
│ │ 海拔     │ 暂不实现   │ 影响大但 averageAltitude 字段不存在，预留接口    │ │
│ ├──────────┼────────────┼──────────────────────────────────────────────────┤ │
│ │ 数据质量 │ 新增置信度 │ 影响综合跑力权重，不改变单次值                   │ │

│ 二、单次跑力优化                                                             │
│                                                                              │
│ 2.1 新增 VdotResult 结构体                                                   │
│                                                                              │
│ 返回结构化结果而非单一                                                       │
│ Double，包含：vdot、confidence、rawVdot、environmentalAdjustmentMinutes      │
│                                                                              │
│ 2.2 体感温度（温度+湿度）                                                    │
│                                                                              │
│ - 温度 ≤ 20°C 时湿度影响极小，直接用实际温度                                 │
│ - 温度 > 20°C 时用简化 Steadman 公式算体感温度                               │

│ - 安全上限：时间调整不超过 10%（防止极端温度导致异常修正）                   │
│ - 向后兼容：humidity 为 null/0 时退化为纯温度行为                            │
│                                                                              │

│                                                                              │

│ - 距离可靠性 (0-0.4)：5-42km 最高，<2km 极低                                 │
│ - 时长可靠性 (0-0.3)：25-120min 最高                                         │
│ - 心率区间适配性 (0-0.2)：T/I 区间最高，恢复跑最低                           │
│ - 环境数据完整性 (0-0.1)：有温度数据更可靠                                   │

│ 三、综合跑力优化                                                             │
│                                                                              │
│ 3.1 异常值检测（Modified Z-score + MAD）                                     │
│                                                                              │
│ - 用中位数和 MAD（中位绝对偏差）替代均值/标准差                              │
│ - MAD 对异常值本身具有鲁棒性                                                 │
│ - Modified Z-score > 3.0 标记为异常值并排除                                  │
│ - 数据量 < 5 时用 fallback madScale = 5.0                                    │
│                                                                              │
│ 3.2 基于实际天数的时间衰减                                                   │
│                                                                              │
│ - 当前问题：用序号 (0,1,2...) 做衰减，一天跑两次 vs 间隔30天的两次跑无区别   │
│ - 改进：用 daysDiff = (当前日期 - 记录日期) / 毫秒每天                       │
│ - λ = 0.03/天，半衰期 ≈ 23 天                                                │
│ - 效果：1天前权重0.97，7天0.81，30天0.41，60天0.17                           │
│                                                                              │
│ 3.3 置信度加权                                                               │
│                                                                              │
│ - 每条记录的总权重 = recencyWeight × confidence                              │
│ - 近期高质量跑步（如10K比赛）权重远高于近期短距慢跑                          │
│ - 历史记录无 confidence 数据时默认 0.5                                       │
│                                                                              │
│ 3.4 稳定性钳制（maxDelta = 3.0）                                             │
│                                                                              │
│ - 综合跑力单次更新变化不超过 3.0                                             │
│ - 真正的体能提升 3+ VDOT 需要数周，钳制会自然释放                            │
│ - 首次计算（无历史）跳过钳制                                                 │
│                                                                              │
│ 3.5 边界情况                                                                 │
│                                                                              │
